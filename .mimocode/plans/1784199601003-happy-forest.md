# Fix: App Crash on Done Button

## Problem
When clicking "Done" after scanning, the app crashes because:
1. `lastResult` is set to null while `AnimatedVisibility` is still rendering
2. `lastResult!!` (non-null assertion) crashes when value is null
3. State updates happen simultaneously, causing race condition

## Solution
1. Use safe call `?.let` instead of `!!` for null safety
2. Set `showResult = false` first, wait for animation, then clear data
3. Or simply check null before accessing

## File to Fix
- `app/src/main/java/com/barcodereader/ui/scan/ScanScreen.kt`

## Changes
1. Line 314: Change `lastResult!!` to safe call with null check
2. Done button: Set showResult=false first, then clear data after delay
