# Fix: APK Installation "Package Invalid" Error

## Root Cause

The `build.gradle.kts` has **no `signingConfigs` block** for the release build type. This means:
1. `gradle assembleRelease` produces an **unsigned APK**
2. We then manually signed it with `jarsigner` + `apksigner` externally
3. The external signing may have corrupted the APK or created conflicts
4. The `.idsig` file (signature dump) was also created alongside

## Fix

Add proper signing configuration to `build.gradle.kts` so Gradle signs the APK during build.

### File: `app/build.gradle.kts`

Add a `signingConfigs` block and link it to the release build type:

```kotlin
android {
    // ... existing config ...

    signingConfigs {
        create("release") {
            storeFile = file("release.keystore")
            storePassword = "barcode123"
            keyAlias = "barcode"
            keyPassword = "barcode123"
        }
    }

    buildTypes {
        release {
            signingConfig = signingConfigs.getByName("release")
            isMinifyEnabled = true
            isShrinkResources = true
            proguardFiles(...)
        }
    }
}
```

### Steps:
1. Move `release.keystore` to `app/` directory
2. Add `signingConfigs` block to `build.gradle.kts`
3. Link `signingConfig` to release build type
4. Add `release.keystore` to `.gitignore`
5. Clean and rebuild: `gradle clean assembleRelease`
6. Delete old GitHub release
7. Upload new properly-signed APK

### Why This Fixes It:
- Gradle signs the APK **during build** (not after)
- Uses v2/v3 signing natively
- No external tools needed
- Clean, single signing pass
- No corruption from multiple signing tools

## Verification
- `apksigner verify --verbose app-release.apk` should show v1+v2+v3
- APK installs on Android device without errors
- No "package invalid" message
